## 1. Цель работы
Целью данной работы является изучение основ разработки мобильных приложений в среде Android Studio, ознакомление со структурой проекта, освоение различных типов контейнеров разметки (Layout) и реализация механизмов взаимодействия пользователя с интерфейсом на языке Java.

## 2. Стек технологий
*   **Среда разработки:** Android Studio Ladybug 2024.2.2.
*   **Язык программирования:** Java.
*   **Минимальная поддерживаемая версия:** API 26 (Android 8.0).
*   **Используемые компоненты:** androidx.constraintlayout, View.OnClickListener, ViewCompat.

## 3. Выполнение работы

### Глава 1-2. Настройка среды и создание структуры проекта
В ходе выполнения первых этапов была произведена конфигурация Android SDK и создание базового проекта. В соответствии с требованиями методических указаний, работа была организована по модульному принципу: каждое задание реализовано в отдельном модуле в рамках единого проекта.

> **[ВСТАВИТЬ СКРИНШОТ: Окно Android Studio со структурой проекта, где видны модули в левой панели]**

### Глава 3-4. Проектирование пользовательского интерфейса
Были изучены принципы работы основных визуальных компонентов (TextView, Button, CheckBox, ImageView). Была реализована сравнительная практика использования различных контейнеров:
*   **LinearLayout:** для последовательного размещения элементов.
*   **TableLayout:** для формирования сеточной структуры.
*   **ConstraintLayout:** для построения сложных интерфейсов с использованием относительных привязок.

### Глава 5. Реализация адаптивной верстки (Ориентация экрана)
Произведена настройка приложения для работы в портретном и альбомном режимах. 
*   Создан альтернативный файл ресурсов в директории `res/layout-land`.
*   Устранена критическая ошибка `NullPointerException` при повороте экрана путем синхронизации идентификаторов (ID) корневых элементов в различных конфигурациях разметки (использование `android:id="@+id/main"`).

### Глава 6-7. Программная обработка событий
В модуле `ButtonClicker` реализована бизнес-логика приложения. В соответствии с заданием, были применены два метода регистрации обработчиков событий:
1.  **Программная установка слушателя** через анонимный класс `View.OnClickListener` для кнопки `btnWhoAmI`.
2.  **Декларативное назначение** через атрибут `android:onClick` в XML-файле для кнопки `btnItIsNotMe`.

Программа успешно выполняет изменение текстовых данных, управление состоянием элементов `CheckBox` и вывод информационных сообщений `Toast`.
